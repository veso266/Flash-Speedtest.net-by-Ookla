Ookla Hosted Reporting

Saoud - November 30, 2017 02:44

All licensed NetGauge applications include access to Ookla’s Hosted Reporting Portal to provide easy access to the results from your installation. In addition to your delivery of your NetGauge installation package your Account Executive will also provide login credentials to access Ookla’s Hosted Reporting Portal. By default every licensed NetGauge application is configured to pass the results at the end of a test via an HTTP Post to http://api.ookla.com/. These test results are not shared by Ookla or included in the test results licensed via our Speedtest Intelligence data product.

Many clients install NetGauge in a local environment that prevents the results from being passed to our database or simply prefer to store results locally in their own database. More information on storing results locally can be found below:

Please note: the Client Location and ISP fields are not present in local reporting, as Ookla uses Maxmind's GeoIP service to resolve this data.

Your result view for your installation will depend on the amount of licensed servers, below is a view from the demo Reporting Portal that stores results from all of our demo tests found at https://www.ookla.com/demo. 

When using a multiple server deployment each server will have an assigned server ID in the settings file. It is important to keep this ID assigned to server as this will determine how we map the results from your test. In the below example results from server ID = 2 would be represented as Las Vegas in the results portal. Please contact your Account Executive prior to changing any server names or IDs.

<servers>

<server name="Atlanta" id="1" lat="33.7483" lon="-84.3911">

<method type="tcp" host="atl.mycompany.com:8080"/>

<server name="Seattle" id="2" lat="47.609" lon="-122.333">

<method type="tcp" host="sea.mycompany.com:8080"/>

</server>

</servers>

The available columns in the reporting portal will populate depending on your test configuration. In the event we do not receive data from a testing user for a particular test component after a completed test, these columns will be hidden from view. This is handled on a per-page level within the hosted reporting. For example, starting with all tests active you will see all columns, if the Packet Loss test and Firewall test were disabled and you you were using a 50 test view after the 50th test these columns would not appear. In viewing historically these columns will then reappear.
