NetGauge: Linux & Unix Server Daemon Installation

Team Ookla - January 23, 2017 08:47

There are two options to install and control the NetGauge OoklaServer daemon.

  • Using a Shell Script (This is the recommended default option for Linux/Unix-Based Servers)

  • Manually via Terminal (Windows Servers)

 

Using a Shell Script (Linux/Unix only) 

Download the install script:

Run this command from your desired directory to download the install script:

curl -O http://install.speedtest.net/ooklaserver/ooklaserver.sh

 

Setup:

Update file permissions to allow installation:

chmod a+x ooklaserver.sh

How to use the installation and management script:

Use the following to view the Help prompt for Ookla Net Server:

./ooklaserver.sh -h

Installation & Startup:

./ooklaserver.sh install

Upon confirmation of installation the shell script will start the Ookla Net Server installation.

 

To manage Ookla Net Server, the following commands are available:

./ooklaserver.sh start
./ooklaserver.sh restart
./ooklaserver.sh stop

 

Completion:

After Ookla Net Server has been successfully installed, verify that Ookla Net Server is externally accessible on the port it was configured for (the default is 8080). Note that both TCP and UDP port 8080 are required to be accessible. Open a browser and navigate to the host server's external IP or hostname on port 8080.

For Example:

http://sea.speedtest.net:8080
http://173.208.32.122:8080

A properly configured installation will return the following message:

OoklaServer
It worked!

If Ookla Net Server is unavailable, please check the following:

  • Ensure that no firewall or router is blocking access. The port must be open and accessible from the Internet.
  • Verify that no other services or applications are using the port.
  • Confirm Ookla Net Server is running:
ps -ef | grep OoklaServer

 

Optional: Edit OoklaServer.properties to change server settings

If desired, you can go back into the folder where you placed the server contents and edit the configuration file OoklaServer.properties. On Windows, edit this in a plain text editor such as Notepad (not Wordpad).

Here are some of the options available:

Name

Description
 OoklaServer.tcpPorts

comma-separated list of TCP ports the server will listen to. By default we use 5060 and 8080, but this can be changed to any TCP ports.
 OoklaServer.udpPorts

comma-separated list of UDP ports the server will listen to. By default we use 5060 and 8080, but this can be changed to any UDP ports.
 OoklaServer.maxThreads 

The maximum number of concurrent threads allowed by the server. This will limit the number of concurrent connections. Note: a client using 8 threads would use 8 connections
