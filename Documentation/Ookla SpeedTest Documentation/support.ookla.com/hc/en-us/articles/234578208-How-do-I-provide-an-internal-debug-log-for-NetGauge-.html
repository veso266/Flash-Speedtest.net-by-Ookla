How do I provide an "internal debug log" for NetGauge?

Team Ookla - November 11, 2016 19:21

If your NetGauge is not providing the results you are expecting (inaccurate results, speeds too slow or too fast), the first place to start is by verifying the Client Configuration. The next step would be to test out a different Configuration Template. We may ask you to provide a "debug log". special troubleshooting output you can access through the Application interface itself. This will often help us assist you with your issue or address a concern you may have. 

The internal debug log contains a lot of data, all generated by the test while it is running. This provides us with visibility, a "window" into the inner workings of the Application.

Obtaining an internal debug log is very easy, and you can do it at any time without affecting the server or future operations of the Application on your computer:

  • Before starting a test, click anywhere on the flash window and type 'debug'. You should see a small window pop up. This window will contain the debug information.
  • Start the test. You should see a lot of text showing up in the debug window. Make sure the issue you reported to Ookla happened while the debug window was open.
  • When the test is over, right click on the debug window and click "Select All". Right click on the debug window again and click "Copy". The debug information is now in your clipboard.
  • Next, create a text file and attach it to a Support Request
  • We will contact you as soon as we have thoroughly examined the debug output. Thank you for helping us, help you!
