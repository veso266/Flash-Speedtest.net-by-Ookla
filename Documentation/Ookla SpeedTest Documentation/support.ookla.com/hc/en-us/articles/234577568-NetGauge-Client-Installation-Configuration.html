NetGauge: Client Installation & Configuration

Team Ookla - November 11, 2016 23:25

The NetGauge client is built in ActionScript 3.0 and requires Flash Player 9.

  • First, you will need access to your web directory. The common path for the following platforms:
 Windows  c:\Inetpub\wwwroot\
 Linux / Unix  /var/www/, /home/httpd/, or /usr/local/apache/htdocs/ 
 OSX  /Library/WebServer/Documents/
 Solaris / Sparc    /home/www
  • Extract the archive into your root web directory mentioned above, or a subfolder, such as /speedtest. This location is potentially defined by the license URL.
  • Within this archive, you will see the following content:.
  • root - Flash client  and supporting content (such as index.html)
  • Legacy fallback support files for the Speedtest® contained in the /speedtest directory. (Details here)

 
Note: Having a testing server be the same physical machine as the web server is not an issue.

For the NetGauge Trial Version: Don't forget to replace the default host (mtl.speedtest.net:8080) with your own! Simply edit settings.xml to make this adjustment.

 

The chart below covers provides the definition for each parameter, along with the default setting.

Parameter Download Upload Definition
 testlength 15 15 length of test in seconds
 initialthreads 4 2 the number of threads the test will start with - this number will scaled based on actual speed and threadratio
 minthreads 2 2 the number of threads the test will start with - this number will scaled based on actual speed and threadratio
 maxthreads 32 (max 96) 32 (max 96) the maximum number of threads the test will use
 threadratio 750K (750KB/s ≈ 6Mbps) 750K when the (Speed in bytes per second / thread count) ratio is larger than threadratio, threads will be added. Example: If the speed is currently 8Mbps, the test will scale up to 8 threads with a threadratio of 1M.
 maxsamplesize  5000000 (5M) 1000000 (1M) maximum amount of data requested or sent in one block
 minsamplesize 32000 (32K) 32000 (32K) minimum amount of data requested or sent in one block
 startsamplesize 1000000 (1M) 200000 (200K) size of the blocks the test will send on start
 startbuffersize 1 1 number of blocks the test will send on start
 bufferlength 5000 (5 seconds) 1000 (1 second) amount of data the test will keep in buffer (this is measured in milliseconds, based on current speed)
 packetlength 1000 (1 second) 1000 (1 second) size of the blocks the test will send (this is measured in milliseconds, based on current speed). Note: This will determine the sample size used

 

Line Quality: <linequality packetlosslength="100" packetlosspause="20"/>

packetlosslength the number of packets used for the packet loss test (default = 100)
packetlosspause the time delay, in milliseconds, between each packet send (default = 20)
