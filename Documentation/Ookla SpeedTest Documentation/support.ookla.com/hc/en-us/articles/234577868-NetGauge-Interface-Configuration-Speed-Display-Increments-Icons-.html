NetGauge: Interface Configuration (Speed Display, Increments, & Icons)

Team Ookla - November 11, 2016 19:21

NetGauge comes equipped several options to configure how the speeds measured will be displayed within the interface. Gauge increments, test results and the server name are all customizable to varying degrees. To modify the speed display configuration, simply edit or add the appropriate interface line in your configuration file. Most options outlined in this document can be used simulateneously, just add the desired parameter within the

<interface />

element in settings.xml.

 

Speed Display Configuration

 

Six speed display template options are currently available:

  • Kilobits per second (default) 
    <interface template="kbps"/>
  • Megabits per second 
    <interface template="mbps" />
  • Kilobytes per second 
    <interface template="kbs" />
  • Megabytes per second 
    <interface template="mbs" />
  • Kilobits per second, if speeds are under 1 megabit, and megabits per second if speeds are over 1 megabit. Gauge display is always in kilobits. 
    <interface template="hybrid" />
  • Kilobits per second, if speeds are under 1 megabit, and megabits per second if speeds are over 1 megabit. Gauge display is always in megabits. 
    <interface template="hybridmbps" />

 

Advanced Speed Display Configuration

 

The following parameters modify how speed measurements are displayed:

  • speedprecision: how many digits after the decimal point (generally only necessary when displaying speeds in Mbps). The default value is 2 with Mbps enabled and zero when using kbps. 
    <interface template="mbps" speedprecision="1" />

    Only one digit will be displayed after the decimal.

    Example: 5.4 Mbps

  • displayspeedunit: This will override the unit of measurement text displayed in the NetGauge interface. 
    <interface template="kbps" displayspeedunit="Kbit per second" />

    Example: 8455 Kbit per second

  • The two options can be combined: 
    <interface template="mbps" speedprecision="1" displayspeedunit="Mbit per second" />

    Example: 8.5 Mbit per second

 

Gauge Increment Configuration

 

Two basic options are available to choose your increments:

  • Use a preconfigured template: 
    <interface incrementtemplate="20M" />

    The following templates are available: 10M, 20M, 50M, 100M, 500M, 1000M or 1G. ex: The 10M template will give you a gauge with increments from 0 to 10Mbps.

  • Specify your own increments, in kbps: 
    <interface increments="0,256,512,1000,1500,3000,5000,10000,20000" />

    Tip! Make sure to specify 9 increments in order, starting with 0, or your setting will be ignored.

 

Advanced Gauge Configuration

 

The following settings are also available to modigy the look of your gauge:

SettingDescriptionDefault ValueValue Type
incrementunitsize Font size of the unit displayed next to gauge increments 9 0-25
incrementzero What should be the unit displayed next to the zero? ”” string
incrementkilo What should be the unit displayed next to increments in kilobits? “k” string
incrementmega What should be the unit displayed next to increments in megabits? “M” string
incrementnozero Do not display the zero. false boolean
incrementnoplus Do not display the + on the last increment. false boolean
incrementdisplay Override the display of the gauge increments. not set see below:
Requires 9 values separated by pipes (|)
Example: 0|1MB|DSL|5MB|CABLE|20MB|FIBER|80MB|120MB
binaryprefix Use binary prefix instead of the SI prefix for kilo/mega units. false boolean

The parameters above can be combined with your configuration, as demonstrated in the example below:

<interface incrementunitsize="6" incrementkilo="K" incrementnozero="true" />

Display Server Name

 

For Template NetGauge applications, there is an option to display the server name above the defined server icon. Doing so utilizes the name parameter from the server defined in the settings file:

<servers>
	<server name="My Server Name" id="1">
		<method type="tcp" host="server.mydomain.com:8080" />
	</server>
</servers>

To enable this feature, add displayserver="1" to the interface settings in your configuration file like the following example:

<interface displayserver="1" clienticon="person" servericon="globe"/>
