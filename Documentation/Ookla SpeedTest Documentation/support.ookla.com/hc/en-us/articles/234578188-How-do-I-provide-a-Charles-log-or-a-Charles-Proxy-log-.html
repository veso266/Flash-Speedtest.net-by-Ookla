How do I provide a Charles log or a Charles Proxy log?

Team Ookla - November 11, 2016 19:21

If you are troubleshooting an issue that Ookla cannot reproduce or for a test that is running on a private network, a Charles log will typically allow our Technical Support Group to analyze the HTTP traffic generated by the test, and aid in identifying a solution. A Charles log will not provide insight into into NetGauge TCP Socket Tests, only HTTP Legacy Fallback Tests.

Charles is a cross-platform application that will record all traffic requested by, or sent to, a web browser. It requires installing an application (see: Charles Proxy, below), and it is often the fastest way to troubleshoot issues that generate error messages on the test. For speed issues, we will normally as you to provide an "internal debug log" instead.

  • Charles Proxy is free demo and is available for most common platforms: http://www.charlesproxy.com/download/
  • Once you've installed the application, make sure that it is running, initiate a test with the Ookla Application that has been resulting in the error or issue in question. 
  • Go back to the Charles Proxy application on your desktop.
  • You should see a summary of all content your browser(s) have requested since you began logging.
  • Save the current session of all recorded traffic by clicking on File -> Save. 
  • Create a Support Request (or add to the one you already have open) and include the .chls files you created
  • We will get back to you as soon as possible!

To share your Charles Proxy log, we recommend using something such as Box.net, Cloud App, Dropbox, Google Drive, etc. Alternatively, you can create an archive and host it on a web/ftp server for Ookla to download and review.
