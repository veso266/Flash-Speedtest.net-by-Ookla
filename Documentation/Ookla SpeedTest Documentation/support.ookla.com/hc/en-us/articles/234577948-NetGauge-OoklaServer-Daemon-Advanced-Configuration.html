OoklaServer Daemon Advanced Configuration

Brian Turley - January 05, 2018 20:15

NetGauge's OoklaServer daemon configuration options are set within the OoklaServer.properties file, located in the same directory as the server application. Below is a list of all available settings and a brief description of each. For a detailed explanation of port configuration and enabling IPv6, please refer to KB article NetGauge: Basic Server Daemon Configuration Options.

Note:  Public Speedtest.net hosts require port 8080 to be used.  If your server host is part of the Speedtest.net network, you should not modify ports as this may prevent your host from being listed publically on Speedtest.net.

 

Parameter Enabled (default) Default value Description
OoklaServer.tcpPorts Yes 5060, 8080 Sets the port or ports that the server daemon listens on. This can be changed to any other port if desired. Only a single port is required. For more information see KB NetGauge: Basic Server Daemon Configuration Options.
OoklaServer.udpPorts Yes 5060, 8080 Utilized for packet loss calculation.
OoklaServer.useIPv6 No True Commented out by default. Remove the comment to allow OoklaServer daemon to bind to an IPv6 address. For more information see KB NetGauge: Basic Server Daemon Configuration Options.

OoklaServer.

allowedDomains

 No *.ookla.com, *.speedtest.net Commented out by default, this setting allows you to limit access to your OoklaServer. The default allows all domains, but remove the comment to only allow access from Ookla.com, Speedtest.net, and any other domain added to the list.
OoklaServer.maxThreads Yes 512 Determines the maximum number of concurrent threads allowed by the server.
logging.loggers.app.name Yes Application Do not modify this setting. This is used for logging purposes.

logging.loggers.app.

channel.class

 Yes ConsoleChannel Determines where logging occurs. The default behavior logs information directly to a console window. If logging to a file is desired, change the value to FileChannel.

logging.loggers.app.

channel.pattern 

 Yes %Y-%m-%d %H:%M:%S [%p] %t Modification of this setting is not recommended. This is needed for log formatting.
logging.loggers.app.level Yes information Determines logging verbosity.

.channel.path

ooklaserver.log 

 Only available if the logging channel is set to FileChannel. This setting determines the location of the log file.

Optional Advanced Logging Parameters

The following optional advanced logging parameters only available when logging.loggers.app.channel.class is set to FileChannel. For more details, please see Poco class FileChannel.

ParameterArgument formatDescription

Parameter Argument format Description
logging.loggers.app.channel.rotation Sets the interval at which a new log file is created. Example: logging.loggers.app.channel.rotation = 5 M
never no log rotation
<n> rotate log if file size exceeds <n> bytes
<n> K rotate if file size exceeds <n> Kilobytes
<n> M rotate if file size exceeds <n> Megabytes
[day,][hh:][mm] rotated on specified weekday/time
daily/weekly/monthly every day/seven days/thirty days
<n> hours/weeks/months every <n> hours/weeks/months
logging.loggers.app.channel.archive The suffix applied to an archived log file's filename.
number an automatically incremented number, starting with 0, is appended to the log file name. The newest archived file always has 0.
timestamp a timestamp in the form YYYYMMDDHHMMSS is appended to the log file.
logging.loggers.app.channel.times Defines how the logging system interprets time formats.
utc Rotation times are treated as UTC.
local Rotation times are treated as the local system time.
logging.loggers.app.channel.compress true/false Automatically compress archived log files
logging.loggers.app.channel.purgeAge Archived log files will be deleted after they reach a specified age.
<n> the maximum age is <n> seconds
<n> minutes he maximum age is <n> minutes
<n> hours the maximum age is <n> hours
<n> days the maximum age is <n> days
<n> weeks the maximum age is <n> weeks
<n> months the maximum age is <n> months, where a month has 30 days
logging.loggers.app.channel.purgeCount integer Archived log files will be deleted if the number of log files reaches a given maximum number.
