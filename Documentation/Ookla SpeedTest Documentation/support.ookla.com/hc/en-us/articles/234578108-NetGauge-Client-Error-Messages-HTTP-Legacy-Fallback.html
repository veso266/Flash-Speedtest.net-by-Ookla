NetGauge: Client Error Messages - HTTP Legacy Fallback

Team Ookla - November 11, 2016 19:21

HTTP Legacy Fallback errors can occur for any of the following reasons:

Error Message Title  
Error Message
Download Test Error

Download test returned an error while trying to read the download file.
Upload Test Error

Download test returned an error while trying to read the upload file.
Latency Test Error

Latency test returned an error while trying to read the latency file.

Resolution and Troubleshooting:

Error Message Title  
Resolution
Download Test Error

This error message means that the test was unable to load one of the legacy fallback http download sample fileson the server. The most common cause of this is an error in the server URL in your settings file for method type HTTP. Test to see if you can load one of the smaller sample files (such as random350x350.jpg) in the same directory as the upload script using a web browser.
Upload Test Error

The scripting language (PHP, ASP.NET, ASP or JSP) is not properly setup or the wrong one was chosen. It could also mean that the upload testing script is on a different host than NetGauge. If that is the case, try copying crossdomain.xml to the web root of the testing server to resolve this.

If the upload test starts but then fails with this error then the problem is usually related to a maximum HTTP POST size getting hit.You will need to adjust this to 1073741824 or 10M to allow larger POST requests from testing clients.
Latency Test Error

This error message means that the test was unable to load the legacy latency test file on the server (latency.txt). The most common cause of this is an error in the server URL in your settings file for method type http. Test to see if you can load latency.txt in the same directly as the upload script using a web browser. If hosted on an alternate domain, it is essential to use a crossdomain.xml policy file to allow access.
