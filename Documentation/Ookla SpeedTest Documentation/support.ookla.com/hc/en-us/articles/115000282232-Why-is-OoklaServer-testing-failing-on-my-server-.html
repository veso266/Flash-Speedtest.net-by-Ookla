Why is OoklaServer testing failing on my server?

Michael Jackson - June 06, 2017 03:21

This article covers common reasons you may see failures when using the Server Tester to test your OoklaServer installation. For more information on OoklaServer setup review our installation instructions for both Unix/Linux and Windows.

Incorrect URL

Note the proper format of an OoklaServer URL below. Many users have reported issues which were caused by simply using the wrong URL.

Incorrect:

speedtest.mycompany.com

http://speedtest.mycompany.com

http://speedtest.mycompany.com:8080

Correct:

speedtest.mycompany.com:8080

Connection Refused

If you get a connection refused error it indicates that your server refused the connection to port 8080 or whichever port you were testing. This typically means that a firewall is blocking access to this port but it can also mean that OoklaServer is not running or not bound to that port.

Check that the proper ports are opened in your server or other firewalls between the Server Tester and your server. You should also ensure OoklaServer is running and that is properly bound to the port being tested. You can review port binding in the initial log output when starting OoklaServer.

Wrong Response

An error that the Server Tester got a "wrong response" from OoklaServer indicates that while the tester was able to connect it did not receive the expected response from OoklaServer.

Typically this means that your web server or another service is bound to the port you are testing and OoklaServer is not listening on that port. To resolve this investigate what is bound to that port and change that service's configuration to not bind to the port. You will need to restart both the interfering service and OoklaServer after changing the configuration.

Version Warnings

You may see a warning indicating that your copy of OoklaServer is out of date. New features are regularly introduced to OoklaServer and some of these features may be required for your server to be enabled. Ensure that you upgrade to the latest version.

The latest version of OoklaServer for all platforms is available here. Upgrading simply involves stopping OoklaServer, replacing the binary/executable with the new version, and starting OoklaServer again.

WebSocket Testing

Starting in OoklaServer version 2.4 we implemented WebSocket based testing. If you are using a version previous to 2.4 you may see errors relating to failed WebSocket testing.

We have also seen this occur when HSTS or other security configuration is interfering with connectivity.

Crossdomain Issues

There are a few reasons you may see issues with your OoklaServer crossdomain information. For more information please review our "How do I fix crossdomain issues?" article.
