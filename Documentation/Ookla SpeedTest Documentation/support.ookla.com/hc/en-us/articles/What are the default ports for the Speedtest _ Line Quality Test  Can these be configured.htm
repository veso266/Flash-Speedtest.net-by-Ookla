What are the default ports for the Speedtest / Line Quality Test? Can these be configured?

Team Ookla - November 11, 2016 21:21

The default port for the Throughput Test for both the Speedtest and Line Quality Test is port 80 / HTTP. This may be changed on your web server.

The features of the Line Quality Test - Firewall Test (Optional), Latency/Jitter Test, and Packet Loss Test - are performed over port TCP/UDP 5060 by default.

If TCP/UDP 5060 is not available for use due to a conflict, an alternate port can be used. It is important to note that using privileged ports (below 1024) will require the daemon to be run as root/Administrator. This is why our recommendation is to use a port above 1024. A great alternate port is 8080 as it is commonly open on client networks.

To specify the new ports to be used, the first step is to edit voipcheck.sh or voipcheck.bat, which is included in the server folder of your install package.  For package content details, please visit the following article:

http://www.ookla.com/support/a22044988/Line-Quality-Installation

The following is an example on how to switch both ports to run on 8080.

Linux/Unix- voipcheck.sh 

java -jar VoipServer.jar --tcpPort 8080 --udpPort 8080 > voipserver.log &

 

Windows - voipcheck.bat 

if (%pid%)==() java -jar VoipServer.jar --tcpPort 8080 --udpPort 8080 > NUL

 

The next step is to then edit index.html to change the ports the client tries to connect to:

[lt]param name="tcpport" value="8080"[gt]

[lt]param name="udpport" value="8080"[gt]
