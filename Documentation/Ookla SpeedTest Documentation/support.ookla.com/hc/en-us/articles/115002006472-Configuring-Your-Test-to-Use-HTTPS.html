Configuring Your Test to Use HTTPS

Serena Batten - January 09, 2018 22:48

To configure your Speedtest Custom test to use https, follow these steps:

  1. Log in to your account at account.speedtestcustom.com/login.
  2. If you have not already configured your test interface, follow the configuration steps for the Template or for Custom CSS here.
  3. Embed your test on your https webpage by following the steps in the Embed Your Test on Your Website article.
    1. Change your iframe embed code from "http" to "https"
  4. Register your server(s) using the steps outlined in the Configuring Your Own Server Network article.
    1. Ensure your server(s) are running the newest version of the test daemon, OoklaServer 2.5, which you can download here.
    2. Terminate all running OoklaServer processes, replace the OoklaServer binary executable with the new version, and then start OoklaServer again.
  5. Once the test daemon version is updated, enable SSL by creating a certificate and private key and setting the path to them:
    1. openSSL.server.certificateFile = cert.pem
    2. openSSL.server.privateKeyFile = key.pem
    3. In your OoklaServer.properties file, change the cert.pem and key.pem to your certificate's path and key files.
    4. Restart OoklaServer to load the new configuration.
    5. Note that:
      1. The SSL certificate should be SHA2.
      2. The certificate cannot be self-signed.
      3. The certificate should be configured for your site's URL (myserver.mysite.com), not the Speedtest Custom Test URL (mycompany.speedtestcustom.com).
      4. The certificate file should load the entire trust chain (including intermediaries and root certificates)
  6. Return to your account and click on the Configuration link.
  7. Select the Servers tab.
  8. In the server list, complete the following steps for each server:
    1. Click on the HTTPS icon. 
    2. Check the Enable HTTPS for this server checkbox.
    3. Click Save.
  9. Our system will validate that the server(s) can be reached via an https connection, then the status icon will turn green. Note that it may take up to 2 hours for the system to complete this validation (see also Troubleshooting > Unable to verify HTTPS server configuration)
  10. When all servers have been validated and each status is green, select the Test tab.
  11. Click the Enable HTTPS checkbox.
  12. Click Save & Publish.
  13. Use the View live test link to preview your https test.
