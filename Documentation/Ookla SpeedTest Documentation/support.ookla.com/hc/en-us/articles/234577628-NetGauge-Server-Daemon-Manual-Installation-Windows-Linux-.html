NetGauge: Server Daemon Manual Installation (Windows & Linux)

Team Ookla - November 11, 2016 23:16

 

Using a Terminal - Manual Install

  • Download the archive containing the NetGauge Server Daemon based on your server platform.
    • Windows Zip (.zip) - Contains Windows Server Build Only
    • GNU GZIP (.tgz) - Contains All Server Builds
  • Open the extracted server folder contents included within the Ookla provided or Trial archive and move the appropriate platform's folder to a location that is not web accessible. Rename this folder to a name of your choosing.
  • Create a copy of the file OoklaServer.properties.default and name it OoklaServer.properties.

Navigate to the directory via a terminal/command prompt and run the daemon manually:

Linux / Unix / OSX / Solaris / Sparc ./OoklaServer
Windows OoklaServer.exe

 

Example:

 

./OoklaServer
2011-10-06 22:37:51 [Information] Generating 10485760 bytes of download data
2011-10-06 22:37:51 [Information] Listening on TCP port: 5060
2011-10-06 22:37:51 [Information] Listening on TCP port: 8080

 

After the initial testing process, you will want run the Ookla daemon in the background since closing the terminal will close the Ookla daemon. To run the daemon in the background, perform the following command:

Linux / Unix / OSX / Solaris / Sparc ./OoklaServer --daemon
Windows OoklaServer.exe /help to registerService

 

Windows Only: Once you have registered the OoklaServer.exe as a Service, you will be able to manage it like any Windows Service via Task Manager or Administrative Tools_Services.

Example: Linux / Unix / OSX / Solaris / Sparc

./OoklaServer --daemon

 

Once running you should be able to telnet into port 8080 on the server from a testing client machine. Note that both TCP and UDP port 8080 are required to be accessible but testing UDP port 8080 is not possible. Testing TCP port 8080 will require the telnet client being installed. Telnet is not enabled by default in Windows Vista/7/8. Open another command prompt window, using the example below (lines in bold are case sensitive and must be typed):

 

telnet> open demo.ookla.com 8080
Trying 74.209.160.18...
Connected to ftp.ookla.com.
Escape character is '^]'.
HI
HELLO 2.3 2015-04-24.0629.291745c
QUIT
Connection closed by foreign host.

 

The next step is to go back into the folder where you placed the server contents and you will see the daemon configuration file called OoklaServer.properties. Edit this in a plain text editor such as Notepad (not Wordpad) if you require adjusting the port(s).

Optional: Edit OoklaServer.properties to change server settings

OoklaServer.tcpPorts

comma-separated list of TCP ports the server will listen to. By default we use 5060 and 8080, but this can be changed to any TCP ports.
OoklaServer.udpPorts comma-separated list of UDP ports the server will listen to. By default we use 5060 and 8080, but this can be changed to any UDP ports.
OoklaServer.maxThreads The maximum number of concurrent threads allowed by the server. This will limit the number of concurrent connections. Note: a client using 8 threads would use 8 connections
