How To Install & Submit Server

Michael Jackson - October 31, 2017 06:09

This article provides a general overview for setting up a Speedtest Server.

Installation Steps

1. Ensure your server meets our minimum Server Requirements. 

2. Install HTTP Legacy Fallback using a supported web server.

  • Example HTTP Legacy URL: http://speedtest.mycompany.com/speedtest/upload.php

3. Install OoklaServer Unix/Linux or OoklaServer Windows. Review the OoklaServer changelog.

  • Example OoklaServer URL: speedtest.mycompany.com:8080

4. Verify all tests in the Server Tester pass, review our Using the Server Tester article for help. Review the installation documents above and other server support documents if any tests fail.

5. Submit the Speedtest Server Form, it will take up to 48 hours for your server to be reviewed.

Common Testing/Submission Issues

Keepalive Issues

Update Keepalive to "open" in your web server configuration. This is explained in the following documents for a few popular web servers Apache, IIS 6, IIS 7.

Please review our article "Why is upload testing failing?"

Please review our article "How to fix crossdomain issues?"

404 File Not Found

Please review our article "Why is upload testing failing?"

Connection Refused

If you are seeing a Connection Refused error for OoklaServer please ensure TCP/UDP port 8080 is accessible on your server and that you are using this port in the OoklaServer.properties file. This can also occur when the OoklaServer process is not running.

If you are seeing a Connection Refused error for HTTP Legacy please ensure TCP port 80 is accessible on your server. This can also occur if your web server is not running. 

OoklaServer Outdated

If you get a warning that OoklaServer is not the latest version please follow the same instructions for installation in our OoklaServer Unix/Linux or OoklaServer Windows documents.

Installation requires stopping all running OoklaServer proceses, replacing the existing OoklaServer/OoklaServer.exe binary, and then restarting OoklaServer. You can use the Server Tester again to verify the upgrade was successful.

My submission is being denied because of IP hostnames?

We require all server URLs to use public DNS resolvable hostnames. This is a requirement to support planned future functionality such as IPv6.

Not allowed:

http://192.168.10.10/speedtest/upload.php

Allowed:

http://speedtest.mycompany.com/speedtest/upload.php
