NetGauge: Tutorial - Creating a dynamic test page using JavaScript

Brian Turley - November 11, 2016 19:21

NetGauge natively interacts with JavaScript and provides several customization options for doing so. All sorts of information can be passed from the embedded NetGauge application directly to the browser’s JavaScript handler. The example in this tutorial will leverage this functionality to display dynamic content outside of the Flash container.

This guide will provide a good start for creating your own customized testing portal, but it only scratches the surface of what you can do with NetGauge and JavaScript. For the purpose of this tutorial, advanced JavaScript and HTML skills are not necessary, however,  a basic understanding of JavaScript and the ability to write a simple JavaScript function are strongly recommended. If at any point during the tutorial you get lost, take a look at the finished example files available below. The example files also contain additional comments and explanations that may be helpful.

Getting Started

In this example, we'll build a simple page that either connects the end user with tech support if the test results are slower than expected, or offers to up-sell the end user to a higher service package if the test was faster than expected. The test page will include the following features:

  • A “current status” bar displaying the last completed test
  • An input box for the user to specify their expected download speed
  • A content area that provides the user with recommended actions based on their test results

 

During the tutorial, you will need to make edits to the following three files, all located within the same directory that contains NetGauge.swf

  • Index.html
  • functions_v3.js
  • settings.xml

 

Editing The HTML

We’ll need to add a space on the page for our dynamic content. Open up index.html in an editor and make the following changes.

  1. One line directly below the <body> tag, add <div id="status">Current status: Waiting</div>
  2. On the next line, add a break <br />
  3. Replace the div element abovebefore with the following <div id="abovebefore">Expected download speed: <input type="number" name="targetDown">Mbps</div>

 

Editing the JavaScript

Next we’ll be adding some JavaScript functions. Open functions_v3.js in an editor and make the following changes.

  1. At the end of the existing function test_completed (but still within the function), add the following:

    var downMbps = Math.round(array.download/10)/100;
var expDown = document.getElementById("targetDown").value;
var afterDiv = document.getElementById("aboveafter");

if(downMbps < expDown ){
afterDiv.innerHTML = "Your download rate of " +downMbps + " Mbps was slower than the expected rate of " + expDown +" Mbps. <br/><br/>Click <a href='http://d.pr/abEf'>here</a> to troubleshoot connection issues.";
}

else if (downMbps > expDown){
afterDiv.innerHTML = "Your download rate of " +downMbps + " Mbps was better than expected! <br/> Need an even faster connection? Click <a href='http://d.pr/q826' target='_blank'>here</a> to upgrade your service.";
}
  2. At the very bottom of the file, add the following function: 
    function test_component_complete(testval){
var statDiv = document.getElementById("status");
statDiv.innerHTML = "Current status: " + testval + " test completed successfully."
}

    Editing The Settings

    Next we’ll modify the NetGauge settings to set the level of JavaScript output. Open up settings.xml and make the following changes.

    1.  Find the line <reporting jsreporting="2"/>
    2. Modify the the reporting setting to <reporting jsreporting="3" jscalls="4"/>

    Give it a test!

    Load your page and revel in your work. Try inputting both extremely high and extremely low expected download speeds and notice the different results.
