Installing HTTP Legacy Fallback

Michael Jackson - October 18, 2017 16:59

The HTTP Legacy Fallback files are our previous HTTP based testing engine. This is still required to support users who may have issues testing against OoklaServer due to firewalls, VPNs, or other network limitations.

Installation Overview

1. Download the files required for HTTP Legacy Fallback.

2. Extract the downloaded archive to your web server root directory.

3. Ensure proper server side code support.

4. Ensure your crossdomain.xml file is available. Detailed information is available in our "How do I fix crossdomain issues?" article. Your crossdomain.xml file should be available in your web server root directory as shown below:

  • http://speedtest.mycompany.com/crossdomain.xml

5. Verify installation is working by using the Server Tester. Your HTTP Legacy upload URL should use the following format, note that hostname, path, and file extension may vary:

  • http://speedtest.mycompany.com/speedtest/upload.php

At this point you can continue with the setup and submission of your server.

Where is my web server root directory?

The root path of a web server is the portion that starts after the domain. Within your web server configuration the root path will be defined. This may be something like /var/www/html or C:\Inetpub\wwwroot\.

This means files under /var/www/html on your server will be displayed starting after your server domain. If you placed your HTTP Legacy files under /var/www/html/speedtest/upload.php then the URL http://speedtest.mycompany.com/speedtest/upload.php should work.

Root path: /var/www/html

Extracted archive: /speedtest/upload.php

Public URL: http://speedtest.mycompany.com/speedtest/upload.php

How do I ensure proper server side code support?

There are several upload files included with the HTTP Legacy file download. You will only need to use one of these files depending on your choice of server side language.

File Name Server Side Language
upload.php PHP: Hypertext Preprocessor - Linux / Windows
upload.asp Active Server Page - Typically Windows
upload.aspx Active Server Page Extended - Typically Windows
upload.jsp Java Server Pages - Linux / Windows

You'll need to properly install and configure your chosen server side language to work with your web server. For example, if you are using Apache as your web server and PHP as your server side language you'll need to ensure PHP is installed and properly configured to work with Apache.

Note: We do not provide support for installing server side languages. There are many publicly available articles covering this information.

What is the crossdomain.xml file and how do I get it?

The crossdomain.xml file is a cross-domain policy file. It grants the Flash Player permission to talk to servers other than the one it's hosted on and is required for Flash to use Speedtest servers. More detailed information, including troubleshooting help, is available in our "How do I fix crossdomain issues?" article.

You can download this file here or create it by making a new file with the following contents:

<cross-domain-policy>
  <allow-access-from domain="*.ookla.com" />
  <allow-access-from domain="*.speedtest.net" />
</cross-domain-policy>
