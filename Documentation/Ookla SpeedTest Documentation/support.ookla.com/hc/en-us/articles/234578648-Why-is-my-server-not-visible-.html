Why is my server not visible?

Michael Jackson - November 11, 2017 20:43

You may sometimes not be able to see or use your Speedtest server for various reasons. This article covers some common issues that may cause your server to not be visible.

Server Monitoring

We monitor the functionality of both OoklaServer and HTTP Legacy Fallback. If either service is not working properly your server will be disabled. You can use the Server Tester to verify functionality of both services. For more information see our article "Using the Server Tester."

Server status is monitored hourly from many different IP addresses. Your server must be functional from any public internet IP to prevent service disruptions. It may take up to 1 hour for our monitoring to test your server again.

My server was submitted, why isn't it on Speedtest.net?

After you have finished setting up and submitting your server it will take up to 48 hours for us to review and respond to your server submission. Please be patient and wait at least this long before worrying that your server isn't working.

My server wasn't working but I fixed it, how long until it is enabled?

Server status is checked hourly. It may take up to an hour before your server is enabled again after a problem is fixed. Note that due to our use of a CDN it may take slightly longer than an hour for changes to propagate.

My server isn't showing up in the city server list?

Speedtest.net only displays 5 hosts per city. This list is determined by:

Uptime Score: how stable a host is over time.
Performance Score: the average results compared to the 5 closest hosts.
Age Score: how long your host has been available for.

Your server's scores are updated often. To get the best score ensure your server stays online and offers the best performance in your region.

My server isn't visible in the mobile device applications?

Our mobile applications require that HTTP Legacy be able to accept POST requests of up to 10M. In the event your server is unavailable on our mobile platforms, we recommend checking your max upload configuration in your web server.

Apache with PHP: (DO NOT use mod_security)
Open apache2.conf or httpd.conf
Search for LimitRequestBody and remove that line if it exists
Open php.conf or php.ini
Search for post_max_size and increase it to 10M
Restart Apache (usually /etc/init.d/httpd restart)

IIS6 using ASP:
Open IIS Manager
Right-click on the server name at the top of the tree and choose Properties
Check the first box for Enable Direct Metabase Edit and click the OK button
Open C:\Windows\System32\Inetsrv\metabase.xml with Notepad (NOT Wordpad)
Find AspMaxRequestEntityAllowed and change it to 1073741824
Find AspEnableChunkedEncoding and set it to False

Once your installation has been updated, please contact Ookla Server Support to refresh your server's Max Upload as this requires a manual check.
