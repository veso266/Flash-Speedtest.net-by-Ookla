Basic Line Quality Test Configuration

The information found here gives a summary of what is contained in both HTML and the settings file which controls the Line Quality Test. For more detail on customizing the behavior of the Line Quality Test see this page.

Setting the Server URL and Host

  1. Open the settings.* file on your web server in a text editor (on Windows be sure to use Notepad and not Wordpad).

  2. Look for the line that starts with <server url=. It should look something like this:

    <server url="http://lqtest.mydomain.com/speedtest/upload.php" linequality="lqtest.mydomain.com" />
  3. Confirm that the URL and host entries match the one noted during installation. If they do not then change them to be the same.

Settings File Parameters

Below are the basic configurable parameters found in the settings file. Advanced tweaking of these settings and others can be found here.

<download testlength="10" threads="2" initialtest="250K" mintestsize="250K" maximagesize="40M" />
testlength number of seconds download test should last (default = 10)
threads 1 for a single download thread or 2 for two parallel threads (default = 2)
initialtest amount of data that will be used for the pretest (default = 250K)
mintestsize smallest amount of data that will be used for the download test (default = 250K)
maximagesize largest sample file that will be used for the download test (default = 40M) 
<upload testlength="10" ratio="10" threads="2" mintestsize="32K" maxchunksize="512K" maxchunkcount="100" disabled="false" />
testlength number of seconds upload test should last (default = 10)
ratio assumed ratio of upload speed to download speed (default = 10)
threads 1 for a single upload thread or 2 for two parallel threads (default = 2)
mintestsize smallest amount of data that will be used for the upload test (default = 32K)
maxchunksize largest block of data to send at a time during upload test (default = 1M)
maxchunkcount maximum number of blocks of data to send (default = 100)
disabled set this to true to completely disable the upload test (default = false) 
<latency testlength="10" waittime="50" />
testlength number of samples to use for calculating TCP latency (default = 10)
waittime number of milliseconds to wait between each TCP request (default = 50) 
<extras autostart="false" repeat="0" allowinteractions="true" />
autostart true would make the test start on page load without needing to press a button (default = false)
repeat the number of seconds between the test repeating indefinitely (default = 0)
allowinteractions false would disable Begin Test and Restart Test buttons, should only be used when autostart is enabled(default = true) 
<errors linktodocs="true" showexpirewarning="true" />
linktodocs false would disable the Documentation link during error messages displayed within the testing application (default = true)
showexpirewarning false would disable the renewal warning displayed 15 days and sooner until the license expires, or an updated license key is entered (default = true) 
<linequality enablefirewall="true" detailedfirewallresults="true"/>
enablefirewall false would disable the firewall test component (default = true)
detailedfirewallresults false would change to a simple presentation of the firewall results shown at the end of test (default = true)

HTML Controlled Settings

Some parameters reside outside of the settings file. These can be found in the source HTML of the web page (either index.html or debug.html in your install package). When changing the port parameters you need to make sure that they match what the server is listening on. Instructions for changing both can be found here.

<param name="tcpport" value="5060">
<param name="udpport" value="5060">
<param name="latencylength" value="10">
<param name="latencypause" value="20">
<param name="packetlosslength" value="100">
<param name="packetlosspause" value="20">
tcpport the TCP port that the Java server daemon is listening on (default = 5060)
udpport the UDP port that the Java server daemon is listening on (default = 5060)
latencylength the number of packets used for the latency test (default = 10)
latencypause the time delay, in milliseconds, between each packet send (default = 20)
packetlosslength the number of packets used for the packet loss test (default = 50)
packetlosspause the time delay, in milliseconds, between each packet send (default = 20)
