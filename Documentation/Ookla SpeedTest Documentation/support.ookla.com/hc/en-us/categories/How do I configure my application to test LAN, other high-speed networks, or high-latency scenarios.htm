How do I configure my application to test LAN, other high-speed networks, or high-latency scenarios?

Team Ookla - November 11, 2016 19:21

Ookla Speedtests come configured with parameters targeting the most common current range of broadband speeds. Currently this is considered download speeds between 1 and 25 Mbps and upload speeds below 10 Mbps. However, you can tweak the test parameters to be reliably accurate up to 100mbps (and improve measurements beyond that).

Please Note: Setting up additional threads does not require any changes to your license key.

To overcome throughput limits caused by response times and the TCP window, the Ookla Speedtest engine supports multiple parallel threads. HTTP standards limit us to two concurrent connections per host. HTTP standards limit us to two concurrent connections per host. To get around this we must provide alternate hosts (or host names for the same machine) for each additional pair of threads. Using http://speedtest.mydomain.com as an example, create up to three additional DNS entries like this:

speedtest2.mydomain.com speedtest3.mydomain.com speedtest4.mydomain.com 

Then edit the server entry in your settings file to include all four hosts like this (using the correct upload script):

<server url="http://speedtest.mydomain.com/speedtest/upload.php"
 url2="http://speedtest2.mydomain.com/speedtest/upload.php"
 url3="http://speedtest3.mydomain.com/speedtest/upload.php"
 url4="http://speedtest4.mydomain.com/speedtest/upload.php" />

To use files from other hostnames, Flash usually needs to receive permission from a crossdomain.xml file on those hostnames. See the What is crossdomain.xml and why do I need it? article for information on how to set that up. 

Once done you will be able to use as many as eight parallel threads (the maximum allowed) for both the download and upload tests. Threads can be either 1 or done in pairs, so the possible values are 1, 2, 4, 6, 8. Use the lowest number that achieves proper speed measurement for your situation! Each additional thread uses more CPU on the client which can impact results on older/slower computers. Eight threads should only be needed when measuring speeds in excess of 100 Mbps.

To increase the download thread count to 8 (or another number) change the threads parameter here:

<download testlength="10" threads="8" maximagesize="40M" />

When testing symmetric connections at very high speeds, as many as 8 threads might also be needed. Here are recommended upload parameters when measure at 100 Mbps or beyond:

<upload testlength="10" ratio="1" threads="8" maxchunksize="1M" maxchunkcount="100" />
