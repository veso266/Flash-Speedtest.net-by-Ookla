Setup takes just minutes... Get started now!

First, you will need to access to your web directory. Common paths include:

Windows Servers c:\Inetpub\wwwroot\
Linux Servers /var/www/ or /home/httpd/ or /usr/local/apache/htdocs/

  1. UnZip Package: Unzip the Trial Package either in your root web directory mentioned above, or a subfolder, such as /speedtest.

  2. Upload Test Configuration:

    1. First find the installationurl line in the included settings.xml file.

      Replace this line with a full URL pointing to one of the included upload scripts. These scripts are located in the speedtest subfolder where you extracted the Trial Package (Step 1). You only need one of these files (upload.php, .aspx, .asp, or .jsp) depending on your Server OS. Note: Please contact your System Administrator if you are not sure. It is impossible for Ookla to access or help you with this important setting.

    2. Once you know the appropriate upload file to use, open settings.xml in a plain text editor and enter the installationurl line to point directly to that file:

      Example: http://demo.ookla.com/speedtest/upload.php

    3. If your web domain is www.mycompany.com and you Unzip the Trial Package into a directory called /apps, then replace installationurl with www.mycompany.com/apps to make the line read:

      <server url="http://www.mycompany.com/apps/speedtest/upload.php" />

      Note the upload.php portion would be if your server uses PHP, otherwise it would be one of either .asp, .aspx, or .jsp.

      To Repeat: If you want to use ASP.NET, ASP, or JSP instead of PHP, you simply need to adjust the installationurl line to end with upload.aspx, upload.asp, or upload.jsp. So for an ASP.NET server you enter:

      <server url="http://www.mycompany.com/apps/speedtest/upload.aspx" />

  3. Test your changes in Step 2: Visit the above URL using your web browser to ensure it is correct and working. You should see: size=500 or very similar. That is the best sign that the script (installationurl setting) is correct and working properly. If it is not working, you need to again verify that the scripting language is properly configured on the server. If you see the source code of the upload script itself, or an error of any kind, then something is not setup right with the chosen scripting language on your web server.

    You must contact your Systems Administrator for assistance with the above. Ookla is not able to assist with these issues for security, liability and practical reasons.

  4. You’re done!: Visit the page by entering the URL in a Web Browser. Use example as shown above (http://www.mycompany.com/apps/speedtest/index.html) and replace the domain name and path to match your own.

    Strictly optional: You can set the Trial Version to a white or black background using the corresponding HTML file provided in the Zip.

  5. Too easy? Good job! You now have a fully functional Trial Version of the Ookla Speedtest operating on your own Server. Perhaps the best news is, the production version setup is exactly the same as above.
